# 🚀 全栈搭建 Waline 评论系统权威指南 (Vercel + Supabase + GitHub)

本教程基于真实的排错与成功部署案例编写。采用 **Vercel（无服务器后端） + Supabase（PostgreSQL 云数据库） + 独立子域名 + GitHub OAuth（免密提权管理）** 的现代化架构。

该方案彻底抛弃了早期 LeanCloud 的束缚，具备**零冷启动延迟、数据 100% 自主掌控、国内跨域解析极速稳定**三大核心优势。

---

## 🛠️ 核心架构与前期准备

在正式动手前，请确保你已拥有以下基础设施账号：
1. **GitHub 账号**：用于后续的超级管理员免密登录。
2. **Vercel 账号**：用于托管 Waline 后端 Server 程序（推荐直接用 GitHub 登录）。
3. **Supabase 账号**：提供完全免费且强大的云端 PostgreSQL 数据库（推荐选择 `Singapore` 新加坡节点，国内直连延迟极低）。
4. **个人域名**：用于绕过 Vercel 默认域名（`*.vercel.app`）在国内的网络污染。本教程以 `comment.yourdomain.com` 为例。

---

## 📅 第一阶段：Supabase 数据库建表与通电

Waline 需要一个强壮的底层数据库来存储评论、计数和用户信息。

### 1. 创建数据库实例与获取密钥
1. 登录 [Supabase](https://supabase.com/)，点击 **`New Project`**。
2. 填写项目名称（如 `my-comments`），设置一个高强度的数据库密码（**务必找个记事本存下来，后面马上要用**）。
3. Region 选择 **`Singapore`**，点击创建，等待约 2 分钟完成初始化。
4. 进入项目后，点击左侧底部的 ⚙️ **`Project Settings`** -> **`Database`**。
5. 找到 **`Connection string`** 区域，切换到 **`URI`** 标签，复制那串以 `postgresql://` 开头的链接。
   * **⚠️ 核心注意**：复制下来后，请手动把链接里的 `[YOUR-PASSWORD]` 替换成你刚才设置的真实密码。

### 2. 绕过 RLS 强制执行建表脚本
1. 点击 Supabase 左侧菜单的 **`SQL Editor`**（`>_` 图标），新建一个查询（New query）。
2. 在浏览器中打开并复制 👉 [Waline 官方 PostgreSQL 建表脚本](https://raw.githubusercontent.com/walinejs/waline/main/assets/waline.pgsql)。
3. 将代码完整粘贴到 SQL 编辑器中，点击绿色的 **`Run`** 按钮。
4. **🔴 关键排坑**：此时系统必定会弹窗警告“Potential issue detected”（未启用行级安全 RLS）。由于评论系统需要允许全网匿名读者读写，且所有防刷限流逻辑已由 Waline 后端代码接管，此处**必须点击中间褐色的 `Run without RLS`**。
5. 看到下方提示 `Success. No rows returned`，说明 `wl_comment`、`wl_users` 等核心数据表已完美落建。

## 🌐 第二阶段：Vercel 后端部署与环境变量

### 1. 部署 Waline 服务端
1. 前往 Waline 官方文档，点击 **Vercel 一键部署** 按钮，将代码 Fork 到你的 GitHub 并导入 Vercel。
2. 设置项目名称（如 `waline-backend`）。

### 2. 注入数据库灵魂
在部署配置页面的 **`Environment Variables`**（环境变量）处，添加以下关键变量：
* **NAME**: `DATABASE_URL`
* **VALUE**: 粘贴你刚才在 Supabase 获取并替换好密码的 **URI 数据库连接串**。

点击 **`Deploy`**，静待 1 分钟。当满屏放烟花并显示 `Ready Latest` 时，后端核心已启动。

---

## 🔗 第三阶段：独立子域名的 CNAME 终极解析

默认的 `vercel.app` 在国内极不稳定，必须绑定自定义域名。

### 1. Vercel 侧分配专属节点
1. 在 Vercel 项目面板，点击顶部 **`Settings`** -> **`Domains`**。
2. 输入你规划的子域名（如 `comment.xfhub.com`），点击 **`Add`**。
3. **🔴 关键排坑**：Vercel 近期进行了 IP 扩容，不要再使用烂大街的通用 CNAME。请仔细查看 Vercel 域名卡片上橘黄色的 `DNS Change Recommended` 提示，复制系统为你**专属分配的哈希 CNAME 地址**（例如：`28673b634ca89203.vercel-dns-017.com`）。

### 2. 域名解析后台（宝塔 / 阿里云 / Spaceship）
1. 打开你的域名解析面板，添加一条新记录：
   * **记录类型**：`CNAME`
   * **主机记录**：`comment`
   * **记录值**：粘贴刚才复制的专属哈希地址。
2. 保存后，回到 Vercel 刷新，等待 1-3 分钟。卡片变绿（`Valid Configuration`）且 SSL 证书自动下发，即代表域名彻底打通。

## 👑 第四阶段：GitHub 终极提权与管理端接管

为了摆脱繁琐的账号密码，我们将直接使用 GitHub 进行后端权限接管。

### 1. 触发 GitHub 快捷生成
1. 浏览器访问你刚才绑定的独立域名（如 `https://comment.xfhub.com`）。
2. 在空白的评论输入框下方，点击 **GitHub 图标（小猫头鹰）** 进行授权登录。
3. 此时，你的 GitHub 头像会出现在左侧，但在数据库底层，你现在的身份仅仅是个 `guest`（访客）。

### 2. Supabase 底层提权手术
1. 回到 **Supabase** 控制台，点击左侧 **`Table Editor`**（表格编辑器）。
2. 打开 **`wl_users`** 表，你会看到刚才通过 GitHub 登录生成的数据行。
3. **双击 `type` 列**：把 `guest` 删掉，手动输入 **`administrator`**，按回车保存。
4. **双击 `url` 列**：改成你个人的博客主页地址，按回车保存。
5. （可选）如果你之前用邮箱盲目注册过，导致表里有冲突的旧数据，请勾选多余的行，直接点击顶部的 `Delete row` 物理清退。

## 💣 第五阶段：实战避坑与 F12 终极冲刷法

在部署调试期间，极易遇到“幽灵缓存”引发的灵异事件。以下是必读排错指南：

### 🚨 异常 1：点击删除评论没反应，F12 抓包显示 `401 Unauthorized`
* **病因**：你在数据库里大刀阔斧地改权限删数据，但你的**浏览器前端依然死死锁定了旧的 JWT 身份令牌**。后端拿着旧令牌去新库里查无此人，直接拒收删除指令。
* **解药（强刷 LocalStorage）**：
  1. 在评论页面按 **`F12`** 打开开发者工具。
  2. 顶部切换到 **`Application`（应用）** 选项卡。
  3. 左侧点击 **`Storage`（存储）**，右侧直接点击带扫把图标的 **`Clear site data`（清除网站数据）** 按钮。
  4. 网页按 `Ctrl + F5` 强刷，重新点击 GitHub 登录注入新令牌。此时再删评论，瞬间秒杀。

### 🚨 异常 2：数据库明明改了名字和权限，但历史评论的头像还是蓝色的几何拼图？
* **病因**：Waline 采用了非规范化的 **数据快照机制（Snapshot）**。评论在点击发送的那一秒，当时的头像和昵称就被“刻印”在了 `wl_comment` 表里。无论你怎么改用户表，旧快照都不会跟着变。
* **解药**：属于正常技术现象。直接用你刚拿到的最高管理员权限，把那些带几何拼图的旧测试评论点垃圾桶删掉。重新发一条新评论，你的完美动漫头像和博主徽章就会立刻出现。

### 🚨 异常 3：控制台爆红 `favicon.ico 404` 或 `Uncaught (in promise) Error`
* **病因**：前者是浏览器没找到后端容器的网站图标；后者通常是本地去广告/翻译插件拦截异步请求导致的信道摩擦。
* **解药**：纯粹的背景噪音。对业务逻辑和访客体验为 **0 影响**，请直接无视。

## 🎉 最终接入

至此，后端引擎轰鸣作响。打开你博客前端（Hexo/Hugo/VuePress）的主题配置文件，找到 Waline 配置项，将 `serverURL` 填入你的域名：

```yaml
waline:
  enable: true
  serverURL: [https://comment.yourdomain.com](https://comment.yourdomain.com)  # 替换为你的真实子域名
  lang: zh-CN
